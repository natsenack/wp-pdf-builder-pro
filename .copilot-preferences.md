# Préférences GitHub Copilot - PDF Builder Pro

## Déploiement

### Commande par défaut

**TOUJOURS utiliser le déploiement COMPLET** sauf indication contraire :

```powershell
cd i:\wp-pdf-builder-pro-V2\build
.\deploy-simple-local.ps1 -all
```

**Mode** : COMPLET (-All)  
**Fichiers déployés** : 217 fichiers du plugin (sans vendor)  
**Destination** : `D:\site\wp\wp-content\plugins\pdf-builder-pro`  
**Étapes** :

1. Compilation Webpack (mode production)
2. Git add des fichiers critiques
3. Copie complète de tous les fichiers
4. Vérification d'intégrité post-déploiement
5. Commit Git (local)

### Autres commandes disponibles

- `.\deploy-simple-local.ps1` (sans -all) : Déploiement rapide en local(fichiers modifiés uniquement)
- `.\deploy-simple.ps1` (sans -all) : Déploiement rapide en ligne(fichiers modifiés uniquement)
- `.\deploy-all.ps1` : Déploiement complet avec vendor + remote
- `.\build-react-bundle.ps1` : Compilation webpack uniquement

## Workflow React/TypeScript

### Build system

- **Webpack** : 5.104.1 (production mode)
- **Temps de compilation** : ~220-280ms
- **Bundle principal** : `pdf-builder-react.min.js` (~616 KiB)

### Structure principale

- **Source** : `src/js/react/`
- **Output** : `plugin/assets/js/` et `plugin/assets/css/`
- **Context API** : `BuilderContext` avec hook `useBuilder()`

## Tests utilisateur

Après chaque déploiement concernant des modifications frontend :

1. Recharger l'éditeur WordPress avec **Ctrl+F5** (vide le cache navigateur)
2. Vérifier la console pour les erreurs
3. Tester la fonctionnalité modifiée

## Nonce System

**Action standard** : `'pdf_builder_ajax'`  
**Manager frontend** : `ClientNonceManager.ts`  
**Vérification backend** : `pdf_builder_verify_nonce($nonce, 'pdf_builder_ajax')`

⚠️ **Important** : Toujours synchroniser l'action nonce entre frontend et backend
